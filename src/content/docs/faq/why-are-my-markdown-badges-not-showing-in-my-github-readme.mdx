# Why are my Markdown badges not showing in my GitHub README?

## What matters first

Badges usually fail in a GitHub README because:

1. the badge URL is wrong,
2. the badge image source is private or broken,
3. the Markdown or HTML around the badge is malformed,
4. GitHub sanitizes the HTML pattern you used,
5. or the badge provider is returning something other than a normal image.

The problem is often the badge source, not Markdown itself.

## The most common badge problem: the image URL is wrong

Many badge failures come from:

- malformed Shields URLs,
- missing separators,
- bad query parameters,
- copied example code that was never customized,
- or a badge endpoint that no longer returns a valid image.

If the URL does not return an image directly, GitHub cannot render it as a badge.

## GitHub README badges are still images

A badge in Markdown is usually just an image, often wrapped in a link:

```md
[![Build Status](https://img.shields.io/...)](https://github.com/...)
```

That means every normal image failure can also break a badge:

- bad remote URL,
- blocked or missing asset,
- unsupported HTML wrapper,
- or a source that changed after the README was written.

## HTML can make the problem worse

Some authors try to use:

- custom HTML badge layouts,
- inline styles,
- complex wrappers,
- or unsupported HTML around badges.

GitHub sanitizes HTML aggressively. So a layout that works locally or on a docs site may fail in a README.

## The safest troubleshooting order

Check in this order:

1. open the badge URL directly and confirm it returns an image,
2. confirm the Markdown image syntax is correct,
3. remove custom HTML wrappers and test the simplest form,
4. check whether the source is public and reachable,
5. then inspect whether the issue is provider formatting, not GitHub itself.

That order catches most badge failures quickly.

## The healthiest README badge rule

Use only the badges that add real value.

Keep them:

- public,
- simple,
- image-based,
- and easy to understand without custom HTML.

If the README stops making sense when the badges disappear, the badge layer is too heavy anyway.
