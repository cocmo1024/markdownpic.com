## What this site optimizes

<div class="signal-strip">
	<div>
		<strong>Problem-first content</strong>
		<p>Pages are structured to solve real Markdown, syntax, editor, or publishing decisions rather than inflate word count around basic definitions.</p>
	</div>
	<div>
		<strong>Long-lived search intent</strong>
		<p>Syntax, standards, editor fit, and publishing-system pages decay much slower than trend posts and therefore build stronger compounding traffic.</p>
	</div>
	<div>
		<strong>Standards-first structure</strong>
		<p>The site is designed so people searching for Markdown syntax, examples, and differences can learn the standard, see the exact case, and then move into tooling and workflows.</p>
	</div>
</div>

## How to use MarkdownPic

<Steps>
	1. Start with the syntax page or standards page that matches the exact Markdown behavior you need.
	2. Move into guides when the problem expands from syntax into publishing, export, reuse, or migration.
	3. Use editor and publishing sections when the remaining question is operational fit, not raw syntax.
	4. Open the tool when the output needs to become a long image, polished screenshot, or visual asset.
</Steps>
