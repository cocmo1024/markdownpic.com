The **MarkdownPic Review Desk** exists to keep Markdown pages from drifting into shallow syntax lists or tool-specific assumptions presented as universal truth.

## What gets reviewed

Major pages are reviewed for:

- syntax accuracy,
- whether a claim is actually standard Markdown or only one renderer's behavior,
- whether a portability recommendation is still defensible,
- and whether a new page is genuinely additive inside the existing cluster.
- whether the title, description, internal links, and structured data all describe the same reader task.
- whether the page exists to solve a real Markdown problem rather than to occupy another phrase variation.

## What review is trying to prevent

The review desk is specifically trying to catch pages that:

- present GFM-only behavior as if it were portable Markdown,
- recommend raw HTML without describing the portability cost,
- or create near-duplicate pages around the same syntax edge case.

## When re-review is warranted

Pages should be revisited when:

- renderer behavior shifts,
- a guide starts overlapping with a FAQ or syntax page,
- or the companion tool changes what the best workflow recommendation should be.
