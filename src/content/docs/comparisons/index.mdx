Markdown comparison pages are where real product and workflow intent usually shows up.

This section compares tools and workflows in ways that reflect portability, publishing friction, image handling, AI fit, and long-term content ownership.

## Comparison pages should answer a decision

A useful comparison page should help a reader narrow a real choice, not simply restate feature tables.

The strongest comparison pages should explain:

- which workflow each option is actually built for,
- what migration cost or lock-in comes with the choice,
- how output portability changes,
- and what breaks later if the wrong tool wins for the wrong reason.

## High-intent comparison paths

Start with [Obsidian vs. Typora vs. VS Code](/comparisons/obsidian-vs-typora-vs-vs-code-for-markdown-publishing/) if your team is comparing knowledge-base, writing-first, and repo-first Markdown environments.
