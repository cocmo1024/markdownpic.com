# Why is my Markdown checklist not rendering?

## Quick answer

Markdown checklists usually fail because:

1. the renderer does not support task lists,
2. the syntax spacing is wrong,
3. the list nesting is malformed,
4. the environment supports plain lists but not checkbox rendering,
5. or the content moved into a platform that does not follow GitHub-style Markdown behavior.

The biggest surprise is that task lists are common, but they are not universal Markdown.

## Task lists are not the same as core Markdown

Checklist syntax such as:

```md
- [ ] open issue
- [x] fix bug
```

is widely recognized because of GitHub Flavored Markdown.

That does **not** mean every Markdown renderer will show it as a checkbox instead of plain text.

## The most common syntax mistakes

Even in task-list-aware environments, checklists can fail if:

- the spacing is wrong,
- indentation is inconsistent,
- the brackets are malformed,
- or the item is mixed into a broken nested list.

So the first question is whether the syntax is valid for a normal task list.

## The most common renderer mistake

Many authors test in one place and assume support everywhere.

That fails when the checklist moves from:

- GitHub,
- one docs site,
- or one editor preview

into another renderer that treats task lists as ordinary text.

At that point the issue is not your file alone. It is renderer support.

## The safest troubleshooting order

Check in this order:

1. confirm the checklist syntax is correct,
2. test whether the renderer actually supports task lists,
3. simplify nested lists if needed,
4. compare the behavior in GitHub or another known GFM renderer,
5. then decide whether the checklist is portable enough for the workflow.

That usually tells you whether the problem is syntax or support.

## The healthiest long-term rule

Use Markdown checklists when the target renderer is known and the checklist is mainly for readable presentation.

If checklist state is critical across multiple systems, do not assume Markdown alone should carry that responsibility.

That is where many teams confuse a useful display pattern with a durable workflow system.
