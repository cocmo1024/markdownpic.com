## What matters first

Markdown alt text should describe the image's purpose in context, not mechanically repeat that an image exists. If the image conveys essential information, the alt text should help the reader understand that information. If the image is decorative, the better answer may be to avoid treating it as content in the first place.

## Standard example

```md
![Comparison chart showing Typora, VS Code, and Obsidian ranked by publishing fit](./images/editor-comparison-chart.png)
```

That is stronger than:

```md
![Image of chart](./images/editor-comparison-chart.png)
```

## What good alt text does

Good alt text:

- communicates the useful content of the image,
- stays specific to the surrounding page,
- and avoids filler phrases like â€śimage ofâ€?unless that distinction matters.

The goal is not to describe every visual pixel. The goal is to preserve the content value that would otherwise be lost.

## What poor alt text does

Poor alt text usually falls into one of three patterns:

- too vague to be useful,
- too long and overloaded with unnecessary detail,
- or duplicated from nearby visible captions without adding value.

All three weaken the page.

## How this connects to Markdown workflows

In Markdown systems, image accessibility is often neglected because the syntax is small:

```md
![alt text](image.png)
```

But the small syntax hides an important editorial decision. The alt text is part of the content quality, not a decorative afterthought.

## The export limitation

When Markdown becomes a long image or screenshot, the accessibility story changes because the output may no longer expose alt text the way HTML does.

That is one more reason to keep HTML as the primary, indexable, accessible layer and treat image output as a derivative channel rather than the main knowledge layer.

## Recommended operating rule

- Write alt text that explains the image's purpose.
- Keep it specific but not bloated.
- Do not leave important images with placeholder alt text.
- Do not rely on image derivatives to carry the full accessibility burden of the original page.
