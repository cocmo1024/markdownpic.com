# Why is my Mermaid diagram not rendering in Markdown?

## What matters first

Mermaid is not part of core Markdown.

A Mermaid diagram only renders when the destination platform explicitly supports Mermaid and the code block is written in the expected form.

If either of those is missing, you usually get a plain fenced code block instead of a diagram.

## The most common renderer mistake

The most common mistake is assuming Markdown alone renders Mermaid.

It does not.

A renderer has to recognize a fenced code block like this:

```md
```mermaid
graph TD;
  A-->B;
```
```

If the platform does not enable Mermaid, the diagram will remain plain code.

## The second common problem: the fence is wrong

Mermaid often fails because:

- the language identifier is missing,
- the fence is malformed,
- indentation is wrong,
- or the diagram syntax itself contains an error.

If the fence is not exactly what the renderer expects, it will not switch into Mermaid mode.

## Platform differences matter a lot

Some platforms support Mermaid in specific Markdown surfaces.

Others do not support it at all, or support it only through plugins, themes, or build steps.

That is why a Mermaid diagram can work on GitHub or a docs stack and still fail in a local preview, exported document, or another publishing system.

## Plugins can also be the problem

Mermaid failures are sometimes caused by:

- outdated plugins,
- conflicting Mermaid extensions,
- unsupported Mermaid syntax for the renderer version,
- or preview tooling that is not using the same Mermaid engine as the destination platform.

The issue can be the environment, not the diagram itself.

## The safest troubleshooting order

Check in this order:

1. confirm the fenced block uses the `mermaid` language identifier,
2. test a tiny known-good diagram,
3. check whether the target platform actually supports Mermaid,
4. disable conflicting preview plugins or extensions,
5. then inspect whether the Mermaid syntax is valid for that renderer version.

That order catches most failures quickly.

## The healthiest long-term rule

Treat Mermaid as a renderer feature, not as guaranteed portable Markdown.

If the file has to travel across many tools, expect Mermaid support to vary and test the real destination instead of trusting one preview.
