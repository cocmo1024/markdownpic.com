The **MarkdownPic Editorial Desk** maintains the site's core reference pages on Markdown syntax, CommonMark and GFM behavior, renderer differences, publishing workflows, and Markdown-to-image decisions.

The editorial job here is not to publish endless cheat-sheet fragments. It is to build pages that resolve real Markdown ambiguity, especially when content has to survive more than one renderer or workflow.

## What the desk is responsible for

- choosing the Markdown questions that deserve durable reference coverage,
- explaining where standard syntax ends and renderer-specific behavior begins,
- keeping tool and editor comparisons tied to long-term authoring fit,
- revisiting pages when portability assumptions or publishing workflows materially change.

## Source inputs

Pages are usually built from:

- CommonMark,
- GitHub Flavored Markdown behavior,
- renderer comparisons,
- export and publishing constraints,
- and editorial analysis aimed at portable Markdown authoring.

## What the desk avoids

The desk should avoid:

- copying syntax snippets with no explanation,
- tool praise that ignores portability,
- and thin pages that only rename a renderer quirk.

The review standard is handled through the [MarkdownPic Review Desk](/review-desk/) and the site's [Editorial Policy](/editorial-policy/).
