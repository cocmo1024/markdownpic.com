## What matters first

Markdown does not natively provide collapsible sections.

When collapsible content works in a Markdown page, it usually works because the renderer accepts HTML such as `<details>` and `<summary>`. That means collapsible sections are not a standard Markdown feature. They are a renderer-supported HTML feature.

## Why people look for this

Collapsible sections are useful for:

- FAQs,
- optional detail,
- long technical explanations,
- and keeping dense pages readable.

The problem is that many authors assume there should be a Markdown syntax for them. Usually there is not.

## When details-summary is reasonable

It is usually reasonable when:

- the target renderer is known,
- the feature is helpful rather than mandatory,
- and the page can still be understood if the collapsible behavior degrades.

This makes it a controlled enhancement rather than a structural dependency.

## When it becomes debt

It becomes debt when:

- the source depends on collapse behavior to stay understandable,
- the same pattern must be repeated across many pages,
- or the content needs to move to platforms that sanitize or ignore the HTML.

At that point the healthier answer is usually a component or a docs-system feature.

## The practical rule

Use collapsible sections only when the page still works without them.

That keeps the source readable and makes the feature optional rather than foundational.
