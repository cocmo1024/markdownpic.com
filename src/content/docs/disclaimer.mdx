## Editorial scope

MarkdownPic publishes reference material, comparisons, and workflow guidance about Markdown, editors, publishing systems, and Markdown-to-image operations.

That guidance is intended to be practical, but it does not replace direct testing in your own environment.

## Renderer variation

Markdown is not a single behavior layer. CommonMark, GitHub Flavored Markdown, static site generators, note apps, export tools, and custom renderers all introduce differences.

For that reason:

- a page may describe the safest general approach,
- but you should still test important files in the exact renderer, exporter, or toolchain you use.

## Comparisons and recommendations

Comparisons reflect editorial judgment based on available information, workflow fit, and observed tradeoffs. They are not a promise that one tool will be best for every user or every organization.

## Tool usage

The companion tool is offered as a utility, not as a warranty-backed production platform. You are responsible for your own source content, exports, privacy choices, and publishing decisions.
