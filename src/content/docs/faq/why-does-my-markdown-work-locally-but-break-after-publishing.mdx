# Why does my Markdown work locally but break after publishing?

## Quick answer

Because local success does not prove publishing compatibility.

Markdown often breaks after publishing when:

- file paths become routes,
- heading anchors change,
- raw HTML is sanitized,
- local assets are not copied into the final build,
- or one renderer supported a feature the published renderer does not.

The Markdown source can be valid while the publishing stack still breaks it.

## Local rendering is often too forgiving

A local preview may:

- resolve relative paths from the file system,
- tolerate local assets,
- support editor-specific extensions,
- or render HTML that a published platform later strips.

That creates a false sense of safety.

## The most common failures after publishing

The most common post-publish breaks are:

- links that still point to `.md` files instead of final routes,
- anchor fragments that no longer match generated heading IDs,
- images that existed locally but were not deployed,
- tables, callouts, or task lists that depended on one renderer,
- and raw HTML that the final system sanitizes or ignores.

These are portability failures more than syntax failures.

## The real question to ask

Do not ask only:

**"Is the Markdown valid?"**

Ask:

**"Is this Markdown valid for the publishing system that will own the final page?"**

That question catches far more real-world problems.

## The safest troubleshooting order

Inspect failures in this order:

1. path and route changes,
2. anchor and heading-ID changes,
3. asset copying and image paths,
4. renderer-specific syntax or extensions,
5. raw HTML sanitization or stripped attributes.

That order usually identifies the break faster than rewriting the file blindly.

## The healthiest long-term fix

The best fix is usually not more local hacks.

It is:

- a clear link policy,
- a known extension policy,
- deliberate asset handling,
- and a publishing workflow that treats portability as part of authoring.

Markdown works best when the team knows where syntax ends and publishing rules begin.
