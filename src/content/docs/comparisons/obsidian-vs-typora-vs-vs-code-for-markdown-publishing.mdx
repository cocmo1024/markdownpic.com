## Quick answer

- Choose **Typora** when the priority is a low-friction writing and preview experience.
- Choose **VS Code** when Markdown lives inside a repository, site, or developer publishing workflow.
- Choose **Obsidian** when the real goal is a larger knowledge system that may later feed publishing and reuse.

## What actually matters

Most comparison pages overvalue feature count. For Markdown publishing, the more important questions are:

1. Where does the canonical Markdown live?
2. How cleanly are images handled?
3. Does the workflow lead naturally into publishing?
4. How much scripting or setup can the team tolerate?
5. Will the same Markdown be reused for AI workflows, docs, or screenshots?

## Typora

Typora is strongest when the problem is **writing friction**. It is weaker when the workflow grows more engineered, collaborative, or repo-driven.

## VS Code

VS Code is strongest when the problem is **publishing operations**. It works well for static sites, docs repos, content under version control, and workflows where Markdown is one part of a larger build system.

## Obsidian

Obsidian is strongest when the problem is **knowledge continuity**. It is a strong choice when notes, drafts, and long-term idea networks matter and the same Markdown may later become articles, docs, or AI-ready source material.

## Practical decision rule

If the workflow starts in a knowledge base, Obsidian often wins. If it starts in a site repo, VS Code often wins. If it starts with writing quality and visual calm, Typora often wins.
