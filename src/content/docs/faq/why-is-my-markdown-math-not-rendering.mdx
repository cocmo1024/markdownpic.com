# Why is my Markdown math not rendering?

## What matters first

Math is not a guaranteed built-in Markdown feature.

Math only renders when the destination platform supports a math engine such as MathJax or KaTeX and your expression uses the expected delimiters.

If that support is missing, the formula usually appears as raw text.

## The most common mistake

The most common mistake is assuming Markdown itself renders math.

It does not.

Markdown can carry the expression, but the platform still has to interpret and typeset it.

Without that extra support, `$...$`, `$$...$$`, or math code fences stay visible as plain text.

## Delimiters matter too

Math often fails because:

- inline delimiters are malformed,
- block math is not placed on its own line,
- dollar signs are being interpreted as normal text,
- or the renderer expects a different format than the one you copied.

A valid formula can still fail if the delimiter pattern is wrong for the destination platform.

## Local preview and published output are often different

Math can also break because:

- your editor preview lacks math support,
- a docs site enables math but your local preview does not,
- or one platform supports math while another treats it as plain Markdown text.

That is why the same file can look broken locally and fine after publishing, or the reverse.

## The safest troubleshooting order

Check in this order:

1. confirm the target platform actually supports math rendering,
2. test a tiny known-good inline or block expression,
3. verify the delimiters are correct for that platform,
4. escape dollar signs that are meant to be plain text,
5. then compare local preview with the real destination environment.

That separates syntax mistakes from missing math support quickly.

## The healthiest long-term rule

Treat math as a renderer capability layered on top of Markdown.

If the file must travel across many tools, do not assume math will render everywhere just because it worked once in one environment.
