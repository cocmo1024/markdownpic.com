# Why is my Markdown table of contents not working?

## Quick answer

A Markdown table of contents usually breaks because:

1. the links point to anchors that changed,
2. the renderer does not support the TOC format you expected,
3. a plugin-generated TOC was never rebuilt,
4. heading IDs changed after editing or publishing,
5. or the platform does not define TOC behavior the way your editor did.

The most common mistake is assuming Markdown has one universal TOC syntax. It does not.

## Markdown does not define one TOC standard

Markdown supports headings and links.

It does **not** define one portable table-of-contents mechanism that every renderer must generate the same way.

That means TOCs are usually built with:

- manual anchor links,
- renderer-generated navigation,
- or a plugin or tool that inserts the TOC for you.

Each path breaks differently.

## Manual TOCs go stale easily

A manual TOC fails when:

- heading text changes,
- duplicate headings create different slugs,
- punctuation changes the fragment,
- or the published renderer generates different IDs than the local tool.

That is why the TOC may look correct in source while the links still fail.

## Generated TOCs depend on the renderer

Generated TOCs fail when:

- the site does not support the generator,
- the plugin uses different slug rules,
- the insertion step did not run,
- or the final publishing pipeline ignored the local tool's TOC behavior.

At that point the problem is not only Markdown. It is renderer behavior.

## The safest troubleshooting order

Check in this order:

1. confirm whether the TOC is manual or generated,
2. inspect whether the target headings still exist,
3. compare the rendered heading IDs with the links in the TOC,
4. check whether a plugin or generator actually ran,
5. then inspect whether the publish layer changed the anchors.

That sequence catches most TOC failures quickly.

## The healthiest long-term habit

Use manual TOCs only when the document is stable enough to maintain them.

Use generated TOCs when the publishing system owns heading navigation reliably.

The real risk is not TOC syntax. It is anchor stability over time.
