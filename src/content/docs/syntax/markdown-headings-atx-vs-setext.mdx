## Quick answer

Use **ATX headings** with `#` for almost all modern Markdown work. They are clearer in source, easier to scan, and safer in long documents. Use **Setext headings** only when you intentionally want the old underline style for level 1 and level 2 headings.

## Standard examples

### ATX headings

```md
# Heading level 1
## Heading level 2
### Heading level 3
```

### Setext headings

```md
Heading level 1
===============

Heading level 2
---------------
```

## Why ATX is the better default

ATX headings win in most real workflows because:

- they scale beyond level 2,
- they are easier to edit during restructuring,
- they are easier to read in plain text,
- and they reduce ambiguity when content is copied, diffed, or rearranged.

If your file will live in Git, docs tooling, AI-assisted workflows, or repeated revisions, ATX is usually the better operational choice.

## When Setext still makes sense

Setext headings still work when:

- the document is short,
- you only need a top-level heading and maybe one second-level heading,
- or the source is intentionally styled for older editorial conventions.

But the benefits are aesthetic more than operational.

## Recommended rule

For MarkdownPic readers, the safest rule is:

- default to ATX,
- allow Setext only for deliberate style reasons,
- and never mix both casually in the same document unless the team has a documented reason.

## What heading decisions affect later

Heading style is not just cosmetic. It also influences:

- how easy the source is to restructure,
- how clearly anchor links are derived,
- and how safely the document can be reused in docs systems and exports.
