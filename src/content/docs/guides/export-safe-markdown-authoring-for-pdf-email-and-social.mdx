## Quick answer

Export-safe Markdown starts with one rule: write for **structural clarity first**, not renderer tricks first.

That means:

- paragraphs should still make sense as paragraphs,
- lists should be real lists,
- headings should carry the structure,
- and visual hacks should stay out of canonical source unless they are worth the portability cost.

If the source depends on one renderer's kindness, export will eventually punish it.

## Why export-safe authoring matters

Markdown rarely stays in one place. Teams write once and then:

- publish as HTML,
- export to PDF,
- paste into email,
- convert to screenshots,
- or turn sections into long images and social assets.

The content that survives those moves best is usually the content written with the least renderer-specific fragility.

## The most common export failures

The biggest failures usually come from:

- line breaks that only worked in one renderer,
- wide tables treated like layout systems,
- image sizing hacks that collapse in another environment,
- badge or embed heavy sections that do not survive export,
- and GFM-only features assumed to be universal.

Those are authoring problems before they are export problems.

## A practical export-safe checklist

When a Markdown file is meant to travel, favor these habits:

1. use headings to express structure instead of styling tricks;
2. use explicit paragraphs and lists instead of newline-dependent layout;
3. avoid overloading tables as visual grids;
4. treat raw HTML as a narrow exception, not a default fix;
5. write links and image references so the source still reads well even if the rendering changes;
6. test the final export for any page that matters commercially or operationally.

## Where MarkdownPic fits

MarkdownPic is strongest when the source is already healthy.

If the canonical Markdown is structurally sound, the site can:

- publish HTML for SEO,
- export sections into images for distribution,
- and keep the original content easy to update.

If the source is weak, the image export simply hides the weakness behind a nicer surface.
