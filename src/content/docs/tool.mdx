The **MarkdownPic Tool** is the execution layer for turning Markdown into long images, polished screenshots, and share-ready visual assets.

Open the live tool here: [tool.markdownpic.com](https://tool.markdownpic.com/).

## What the tool is for

The tool is meant for the point in the workflow where the Markdown source is already understood and the task becomes visual distribution.

Typical use cases include:

- turning a section of Markdown into a social share image,
- exporting a long-form snippet as a polished PNG,
- adding a watermark before distribution,
- and building presentation-ready visual assets from clean Markdown source.

## What the tool supports today

Based on the current live interface, the tool currently exposes:

- browser-local auto-save,
- Markdown to PNG export,
- inline image insertion,
- watermark controls,
- and theme presets for visual output.

## What the tool should not replace

The tool should **not** become the canonical place where the only copy of the content lives.

The safer workflow is:

1. keep the source in Markdown,
2. validate the Markdown against the renderer or publishing target,
3. then use the tool when the output needs to become a long image or polished visual asset.

## When the tool is the right choice

Use the tool when:

- the output will be consumed visually,
- layout polish matters more than source portability at that final stage,
- or a section of Markdown needs to become a reusable image for docs, social, or sharing workflows.

Do not expect the tool to solve every portability issue inside the Markdown source itself. That work still belongs in the syntax and workflow pages across the site.
