## Quick answer

Use **inline links** as the default. Use **reference links** when the same URL repeats, when long URLs hurt readability, or when you want easier maintenance. Use **autolinks** only when the renderer supports them and the plain URL form is acceptable.

## Standard examples

### Inline link

```md
[MarkdownPic](https://markdownpic.com/)
```

### Reference-style link

```md
[MarkdownPic][site]

[site]: https://markdownpic.com/
```

### Autolink

```md
<https://markdownpic.com/>
```

## Which form is best

### Inline links

Best when:

- the link appears once,
- readability stays good,
- and the document is short to medium length.

### Reference links

Best when:

- the same destination appears multiple times,
- URLs are long,
- or the document is maintained like source code and link targets may change.

### Autolinks

Best when:

- the literal URL is useful to show,
- or the Markdown is intentionally minimal.

They are not the best choice when you need descriptive anchor text.

## Recommended rule

Use inline links first. Move to reference links when maintenance or readability clearly benefits. Reserve autolinks for cases where showing the literal URL is part of the point.
